15. 三数之和——3sum

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组。

官方解答

方法：排序 + 双指针

一、核心思路（为什么要「排序 + 双指针」？）

  1. 排序的作用：
    • 让相同元素聚在一起，方便「去重」（避免重复输出同一个三元组）；（这里解决了我的疑虑：如果是-4 -1 3 4 8 -1 0 1还是会有重复，因为相同的-1索引离得太远，会有相同的三元组：-1 0 1。但是排完序之后就挨在一起了）
    • 让数组有序，支持「双指针」快速查找目标值（替代三重循环，降低时间复杂度）。
  2. 双指针的作用：
    • 固定一个数 a 后，用两个指针快速找 b 和 c，使得 b + c = -a（因为 a+b+c=0）；
    • 时间复杂度从三重循环的 O (n³) 降到 O (n²)，能通过 LeetCode 的大数据测试用例。

二、完整解法步骤（带代码 + 注释）

三、逐步骤拆解（用例子演示，一看就懂）

用题目样例 nums = [-1,0,1,2,-1,-4] 演示，排序后数组为 [-4, -1, -1, 0, 1, 2]。

步骤 1：排序数组

排序后：[-4, -1, -1, 0, 1, 2] → 相同元素聚在一起（两个 -1），为去重和双指针打基础。

步骤 2：固定第一个数 a（循环 i）

  • 目标：遍历每个可能的 a，找对应的 b 和 c。
  • 去重 1：如果当前 a 和前一个 a 相同（比如 i=2 时的 -1 和 i=1 时的 -1），直接跳过 → 避免重复输出 [-1,-1,2]。

步骤 3：双指针找 b 和 c（以 i=1 为例，a=-1）

  • a = nums[1] = -1 → 目标 target = -a = 1（需要找 b + c = 1）。
  • 初始指针：left=2（nums[left]=-1），right=5（nums[right]=2）。
第一次查找：
  • sumBC = -1 + 2 = 1 → 正好等于 target！
  • 加入结果集：[-1, -1, 2]。
  • 去重 2：left 后面的元素（nums [3]=0）和当前 left（nums [2]=-1）不同，不用跳。
  • 去重 3：right 前面的元素（nums [4]=1）和当前 right（nums [5]=2）不同，不用跳。
  • 移动指针：left=3，right=4。
第二次查找（left=3，right=4）：
  • sumBC = 0 + 1 = 1 → 正好等于 target！
  • 加入结果集：[-1, 0, 1]。
  • 去重 2：left 后面没有元素，不用跳。
  • 去重 3：right 前面没有元素，不用跳。
  • 移动指针：left=4，right=3 → 指针交叉，退出当前循环。

步骤 4：继续遍历其他 a

  • i=2 时，a=-1 和 i=1 时的 a 相同 → 去重 1，跳过。
  • i=3 时，a=0→ 目标target=0，双指针找b+c=0：
    • left=4（1），right=5（2）→ sumBC=3>0 → right–，指针交叉，无结果。
  • 后续 i=4、i=5 时，a 为正数，b 和 c 更大，sum 不可能为 0 → 循环结束。

最终结果：

[[-1,-1,2], [-1,0,1]] → 无重复，符合题目要求。

四、关键细节（为什么要这么设计？）

  1. 排序是核心前提：
    • 没有排序，相同元素分散，无法快速去重；
    • 没有排序，双指针无法通过移动方向调整 b+c 的大小。
  2. 三次去重缺一不可：
    • 去重 1（a）：避免同一 a 对应多个重复三元组；
    • 去重 2（b）：避免同一 a 下，同一 b 对应多个重复三元组；
    • 去重 3（c）：避免同一 a 下，同一 c 对应多个重复三元组。
  3. 双指针移动逻辑：
    • sumBC < target：b 太小 → 左指针右移（找更大的 b）；
    • sumBC > target：c 太大 → 右指针左移（找更小的 c）；
    • 指针交叉即停止：避免 b >= c 导致重复查找。

五、总结

这个解法的核心是「排序 + 双指针 + 去重」：

  • 排序解决「去重困难」和「双指针查找基础」；
  • 双指针解决「效率低下」（从 O (n³) 降到 O (n²)）；
  • 三次去重解决「结果重复」（确保每个三元组唯一）。
